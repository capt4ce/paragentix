PARAGENTIX · REVIEW

Retry a completed job from its detail modal

Smallest safe change: reuse the existing retry API and scheduler. No duplicate job, confirmation dialog, or new dependency.

Modal change

For a done job, show a primary action. Clicking it resets the same job to todo; the scheduler starts a new run and increments the attempt count.

1 · Done
Review timeline and prior attempt.
2 · Retry
Reset the existing job to todo.
3 · Reprocess
Scheduler creates the next run.

Acceptance checks

  • Retry appears only when the job is done.
  • One click invokes POST /api/jobs/:id/retry.
  • The modal closes and the board refreshes.
  • Backend rejects retry from any non-done state.
