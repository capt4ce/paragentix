Design

Clear controls for Jobs and Columns.

Keep Board-level + Add job at the top and add a distinct + Add column tile immediately after the last Column in the horizontal board. Existing per-Column Add job controls remain.

Confirmed placement

  • Top + Add job remains visible, including on an empty Board.
  • + Add column matches the current .newlane pattern and opens the existing Column creation dialog.
  • The tile is the first board item when empty and follows the final Column when non-empty.
  • There is no remaining placement question.

Execution contract

Each Column is a serial execution queue, not a status. Jobs run one at a time and append at the tail. Separate Columns may run concurrently. Status changes never move Jobs between Columns.

Minimal change

Reuse existing Column and Job dialogs, handlers, defaults, and APIs. Preserve per-Column Add job. No backend, database, schema, or API-contract change.

Exact acceptance checks

  • Empty Board: both top + Add job and + Add column are visible.
  • Non-empty Board: + Add column is inside the scroller immediately after the last Column.
  • Activation opens the existing Column dialog exactly once and preserves submit/cancel/error behavior.
  • Native-button semantics, accessible name, visible focus, logical tab order, Enter/Space activation, and a minimum 44×44px touch target.
  • On mobile, the tile remains the final item in the horizontal-scroll sequence; scrolling to the end reveals it without clipping.
  • Top Add job retains empty-Board creation and final-Column targeting tests; per-Column controls remain intact.
  • Queue tests prove no interruption, predecessor terminal-state gating, concurrency across Columns, and no status-based movement.
  • Persisted dark/light theme and responsive review pass.
