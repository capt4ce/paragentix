Job session comments

Add a comment composer to the Job detail modal so a user can reply to and interact with the active CLI session without waiting for the job to become blocked.

Recommended

Behavior

  • Show the composer only while a job has an active session: in_progress or blocked.
  • Send the comment verbatim to the job’s tmux session followed by Enter.
  • Keep the modal open, clear the composer after success, and refresh the timeline.
  • Disable Send for blank comments and while a send is in flight.
  • Show an inline error if the session is unavailable; do not change job state.

Smallest safe backend

  • Add POST /api/jobs/:id/comment with {"comment":"…"}.
  • Reuse the existing authenticated, user-scoped job route and tmux send-keys path.
  • Accept only active states and reject empty/oversized input at the API boundary.
  • Record a comment job event against the active run so the interaction is visible in Timeline.
