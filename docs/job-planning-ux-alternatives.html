The planning pipeline

The AI proposes structure, but the user remains the final authority before execution starts.

01

Describe

User states the desired outcome and constraints.

02

Break down

AI generates an editable todo list with concrete outputs.

03

Organize

Independent tasks become parallel columns; dependencies set order.

04

Approve

User edits, validates, and explicitly accepts the execution plan.

05

Execute

The planning board becomes a live status and recovery surface.

Three interface directions

Conservative · safest

Guided planning wizard

A focused, three-step path from request to reviewed execution plan.

  • Strong protection from accidental starts
  • Clear on small screens
